vim笔记

Author: Arthur Lu Date: August 1, 2019  00:17:18

一、基础

四种模式:

  • Normal
  • Insert
  • Command
  • Visual
    asdf
    基本结构:
  • Buffer
    • vim打开一个文件，会把文件加载到缓冲区
    • 对文件的修改是针对缓冲区的
    • 当:w进行保存时会把缓冲区的内容输出到磁盘
  • Window, 可视化的分隔区域
    • 一个buffer可以划分为多个window
    • 每个window都可以对应不同的buffer
  • Tab, 可以容纳一系列Window的容器
    • 方便管理不同类型的Window

默认打开一个文件，使用一个Tag，其中只有一个Window，并使用了该文件的Buffer。

二、Nomral模式

Normal模式主要是用于阅读文本。
光标移动:

  • hjkl
  • w/W, 移动到下一个word/WORD开头(word是指非字母作为word的分隔符, WORD代表以空格符作为word的分隔符)
  • b/B, 移动到删一个word/WORD开头 woijojoiw
  • e/E, 移动到下一个word/WORD结尾
  • $, 快速移动到行尾
  • 0, 快速移动到行首
  • ^, 快速移动到第一个非blank的字符
  • g_, 快速移动到最后一个非blank的字符
  • ctrl + o, 移动到光标上一个位置
  • ctrl + u, ctrl+o的反向操作
  • ctrl + u, 向上翻页
  • ctrl + f, 向下翻页
  • gg, 移动到页首
  • G, 移动到页尾
  • Ng, 移动到第N行, N是一个数字

搜索:

  • 行内搜索:f{char}, 光标快速跳到匹配的字符。;移动到该行下一个匹配的字符 ,移动到该行上一个匹配的字符。
  • 行内搜索:F{char}, 光标快速跳到上一个匹配的字符。;移动到该行下一个匹配的字符 ,移动到该行上一个匹配的字符。
  • /{word}, 正向搜索word n/N跳转下一个/上一个
  • ?{word}, 反向搜索word

快速删除:

  • x, 删除一个字符
  • dw, 删除一个单词
  • dd, 删除一行
  • dt{chr}, 删除当前光标掉符号chr内的内容
  • d$, 快速删除当前光标到行尾

快速替换:

  • r{chr}, 替换当前光标的字符为chr
  • s, 删除当前字符并进入编辑模式
  • C, 删除当前光标到行尾并进入编辑
  • :%s/origin/target/g, 将全文中的origin替换为target
  • :%s/origin/target/gc, 将全文中的origin替换为target, 会进行提示

1.寄存器

在Visual下通过y/x等进行复制/删除操作，都会将被操作的内容移到寄存器中，默认是无名寄存器。通过p可以将寄存器中的内容复制到光标所处的位置。

通过"{register}来使用寄存器,例如:"ayy, 会将当前行全部复制到寄存器a中。
:reg {register} 可以查看保存在寄存器register中的内容

寄存器分类:

  • 无名寄存器, 默认采用该寄存器。被称为unnamed
  • 普通寄存器.
  • 专用寄存器
  • 系统剪贴板, 需要通过:echo has("clipboard")来确定系统剪贴板是否存在。

3.主题选择

  • :colorscheme 显示当前颜色主题
  • :colorscheme <ctrl+d> 显示所有可以选择的颜色主题

三、Insert模式

从Normal切换到Insert模式的几种方式:

  • i
  • a
  • o
  • I
  • A
  • O
  • gi
  • s, 删除当前字符并进入编辑模式

常用快捷键, 非常方便:

  • ctrl + h 删除上一个字符, 类似于Escape
  • ctrl + w 删除上一个单词(删除直到)
  • ctrl + u 删除一行
  • gi 光标回到上次编辑位置, 并进入编辑模式

上述快捷键同样适用于终端操作.这里补充下终端的操作(不适用于vim):

  • ctrl + a 光标快速移动到开头
  • ctrl + e 光标快速移动到结尾

1.插入补全

在插入模式下可以对当前光标所处的单词后段进行补全，补全的命令为:

  • ctrl + n/p 普通补全, n代表下一个补全 p代表上一个补全
  • <ctrl + x><ctrl + f> 通过目录补全路径名, 例如对 /var/ 可以进一步补全
  • <ctrl + x><ctrl + o> 全能补全 需要插件支持
  • :r! echo % 获得当前文件名
  • :r! echo %:p 获得当前文件绝对路径

四、Visual模式

Visual模式用于进行块选择，并对被选择的块进行操作。

对多行的行首进行操作:

  • <C-v> 选择多行
  • I 跳到行首并进入Insert模式
  • 在行首进行各类操作
  • Esc 点击两次将会把操作应用到被选择的行

对多行的行尾进行操作:

  • <C-v> 选择多行
  • $A 跳到行尾并进入Insert模式
  • 在行尾进行各类操作
  • Esc 点击两次会把操作应用到被选择的行

五、多文件

在第一章中提到了Buffer代表了一个文件的缓冲区，但是在一个Vim进程中是可以打开多个文件的，这样会加载多个文件的Buffer，在不同的Buffer间也可以进行跳转。

多文件以及相关Buffer的相关命令·

  • :e {filename} 加载指定的文件到当前Buffer以及Window
  • :ls 查看所有的缓冲区
  • :b {idx} 跳转到指定缓冲区
  • :bnext 跳转下一个缓冲区
  • :bprevious 跳转上一个缓冲区
  • :bd 关闭当前buffer但是不退出window

多文件及Window相关的命令

  • 一个buffer可以划分为多个window
  • 每个window都可以对应不同的buffer
  • :sp {file} 水平分割, :vs {file} 垂直分隔. {file}可以省略，若省略则默认使用原buffer。
  • ctrl + w + 方向键, 选择当前窗口, 光标会跳转到当前窗口.
  • ctrl + w + 大写HJKL, 将当前window进行上下左右移动

多文件及Tab的相关命令

  • :tabnew 打开一个空白的标签页
  • :tabe {file} 在新标签中打开文件file
  • ctrl + w + T 将当前窗口移动到新的标签页
  • :tabc 关闭当前tab及其所有window
  • :tabo 关闭除当前tab外的所有tab
  • :tabn 切换到下一个标签页
  • :tabp 切换到上一个标签页

六、Text Object

Vim中通常是操作文本对象来完成对文本的删除/变更/选中等。

命令模版:[number] <command> <text object>

  • number, 操作个数, 可选, 默认为1
  • <command>, 操作模式
    • d, 删除文本对象
    • c, 修改(删除后进入insert模式)
    • y, 复制文本对象
    • v, 选中文本对象
  • <text object>, 文本对象
    • w, 删除当前光标到单词尾的内容
    • iw, inner word, 当前光标所指的整个单词
    • aw, around word, 包含单词周边空格
    • i(, 圆括号内的所有内容
    • i[, 方括号内的所有内容
    • i{, 大括号内的所有内容
    • i", 双引号内的所有内容
    • i', 单引号内的所有内容

七、Vim配置

在vim中可以通过set xx和xxx on进行相关的配置，但是都只是在当前vim进程中有效，若希望在全局vim中有效可以配置.vimrc文件。

vim ~/.vimrc, 编辑vim配置文件, 常见配置请看附录部分。

vimrc编辑后的生效方式:

  • :source ~/.vimrc, 通过source选择使用的vimrc文件即可生效。
  • 退出vim后重新进入(必须是~/.vimrc)
  • :source % 若当前已经在vimrc了可以直接通过该命令加载vimrc

1.映射

常用于设置快捷键

基本映射(递归映射)，下面定义了不同模式下的映射:

  • :map {origin} {target} 将normal模式下的origin动作映射为taget动作
  • :unmap {origin} 将normal模式下的origin动作映射取消
  • :[n/v/i]map {origin} {target} 将normal/visual/insert模式下的origin动作映射到target

上述的基本映射都是递归映射, 即当A映射到B, B映射到C时，就会导致A映射到C。为了避免此类递归映射，我们应该使用非递归映射，这也是基本所有人遵循的原则:

  • :noremap {origin} {target} 将normal模式下的origin动作映射为taget动作
  • :unmap {origin} 将normal模式下的origin动作映射取消
  • :[n/v/i]noremap {origin} {target} 将normal/visual/insert模式下的origin动作映射到target

2.特殊字符

  • <space> 代表空格
  • <C-x> 代表ctrl + x
  • <Fx> 代表Fx, x代表一个数字, 例如F2

附录一、常用配置和命令

需要知道, 基本上所有的配置开启和关闭都有等价形式:

  • set xxwx <-----> set noxxx
  • xxx on <-----> xxx off

常用配置项:

  • 语法高亮: :syntax on
  • 搜索高亮: :set hls
  • 实时搜索: :set incsearch
  • 显示行号: :set nu
  • 自动缩进(即回车自动和上一行对齐): :set autoindent
  • :set paste
  • 设置颜色主题: :colorscheme <scheme>

常用命令:

  • 查看所有的配置: :h option-list
