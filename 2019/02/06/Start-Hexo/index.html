Start Hexo

Author: Arthur Lu Date: February 6, 2019  14:06:09

本文总结了在GitHub Pages上搭建和使用Hexo博客平台的经验。
Hexo是什么？借助Hexo官方中文文档的解释：

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

GitHub Pages是什么？借助GitHub Pages官网的解释：

GitHub Pages is designed to host your personal, organization, or project pages directly from a GitHub repository.

另外，GitHub Pages的使用是受限的, 请参考此处。不用担心，一般来说完全足够我们使用了。

一、GitHub配置

1.创建仓库

只需要新建一个仓库，并且命名为.github.io即可。通过URL访问<username>.github.io就是返回的该仓库下的index.html网页。而通过访问URL<username>.github.io/path/file就是返回该仓库下的path/file。

2.配置DNS

<username>.github.io这样的域名不够个性化，可以配置CNAME解析，将其他域名解析至该域名中。但是github并不允许任意域名都能解析过去，需要在仓库中配置CNAME文件，并且在里面输入运行解析的域名。例如：echo "blog.asir.com" >> CNAME。

二、Hexo安装

1.Node.js

Hexo是基于Node.js的基础上开发的，因此使用Hexo前需要安装Node.js。
若安装Node.js成功后，可以在控制台下通过node -v指令查看版本，若可以出现版本信息，则表示安装成功。

2.Hexo

在控制台下通过NPM指令直接安装Hexo:npm install -g hexo-cli，执行完成以后Hexo安装成功，可以通过hexo -v观察版本信息。

2.编写博客

hexo new [layout] <title>
一般而言，layout有三种取值 : post, page, draft。省略[layout]时，默认取值为来自_config.yml中的default_layout属性，该属性的默认值为：post。
title可以用双引号，也可以省略。当博客title中间包含空格时，一定要用双引号。

  • post
    将会在创建文件source/_posts/，并以此作为博客名。
    以该方式的博客，在部署以后，其标题通常会在/archives/index.html中显示出来，其博客内容文件一般存放在///index.html中。
  • page
    将会创建文件source/_post//index.md，通常在博客中添加链接//index.md来进行访问。
  • draft
    用于做草稿的，使用的不多，此处不做介绍。请参考此处。

3.主题

  • 下载主题
    不同的主题将会解析出不同的静态文件来显示博客，选择一个自己喜欢的现成的主题是一个非常棒的想法。从这些主题中选择一个自己喜欢的吧！
  • 切换主题
    这些博客通常提供了自己的GitHub和Demo。将博客从GitHub中下载下来，并解压至themes/文件夹下。在_config.yml中的theme属性中输入自己想采用的主题名。

用一个现成的主题非常简单，但是如果想要自己的博客显得有个性，还需要对在对博客源码进行修改哦(博客源码就在themes/中)。

4.部署

博客和主题准备好以后，就可以进行部署了。

  • 生成
    hexo generate和hexo g，都可以解析和生成静态文件。这些文件将会被放置至/public文件夹中。实则部署的时候，只会上传该文件夹中的内容。
  • 本地模式
    hexo server将会在本地运行一个Node.js服务器，用于在本地显示博客。默认端口4000，通过localhost:4000来访问本地博客。该模式在写博客的时候相当方便，因为可以动态看到编写博客的内容。

  • GitHub

四.其他

1.自定义layout

一切自定义layout文件都和post的行为相同，在source/_posts/<title>中创建。
其实，当执行了hexo new [layout] <title>指令后，首先将会到scaffolds中寻找同名[layout]文件，找到以后将以该[layout]文件的内容创建博客文件。

2.显示照片

若想在博客中显示图片，有两种方案：

  • 传统方案
    新建一个/source/img/文件夹，hexo在生成的时候，会将source文件夹下所有单元生成到/public/根文件夹下。因此在博客中用![](/img/xx.jpg)的方式可以显示照片。当然为了各个博客之间解耦，可能需要在/source/img/下面继续创建博客文件夹。
  • Hexo3方案
    可以在_config.yml中将属性post_asset_folder置为true。以后每次创建新的博客，都会在/source/_posts/下额外生成一个文件，该文件与博客同名，用于放置资源文件。
    需要注意的是，当进行生成静态文件时，博客内容文件将会和资源文件夹下的文件放置在一起。这样只需要通过![](xx.jpg)就可以显示照片了。

    五、FAQ

    1.hexo server无响应

  • 问题:
    执行了指令hexo server开启本地模式后，
通过localhost:4000访问网页，没有网页返回。
  • 解决方案:
    需要注意此时很可能是端口被占用，
可以通过hexo server -p 4001，换一个端口进行测试。
此时访问的URL为localhost:4001

    2.hexo deploy无反应

  • 问题:
    当配置好Git后，执行hexo deploy没有任何返回。
  • 解决方案:
    在执行hexo deploy前，请确保安装了所需要的组件：
    npm install hexo-deployer-git --save
并且在_config.yml的属性对中，冒号后面必须空一格，再写值。
    deploy:
        type: git
        repo: abcdefg
        branch: master
