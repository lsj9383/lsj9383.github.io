funny parameter in c++

小记
2018-08-23

C++作为一门较为古老的语言，缺少类似Python，C#中进行关键字参数传递的机制，只能进行位置传参数、默认值传参以及不定长参数。对于参数较长的情况，直接观察函数调用很难理解其传参语义，如下函数声明和传参数形式，时间久了后，很难去理解Finish(.)函数调用的意义。这里讨论一下如何去构造优美的传参，只做娱乐，不做工程化。

一、构造函数传参数

一个类的构造函数参数过于复杂的时候，可以通过Builder模式结合调用链进行。Builder不仅可以结构化传参，还可以构滞后对象的构造。

这对于构造一个对象非常方便，并且并不会过多影响效率，这对内存的影响是多加了一层属性的换成，对运行效率的影响是多了一层浅赋值。
